The Problem

Fine-tuning a 20GB model usually means saving a new 20GB file. Traditional storage is blind to the structure of your data, leading to massive redundancy and slow transfers.

The Solution

vekt decomposes `.safetensors` into individual layers. It hashes every tensor, stores them in a content-addressable store, and reconstructs models from lightweight JSON manifests.

How it works

It feels just like Git, but built for tensors.

vekt init
Initialized empty vekt repository in ./.vekt
vekt add llama-3-8b.safetensors
Scanning tensors...
Hashing [################################] 100%
Saved 291 tensors. Manifest written to llama-3-8b.vekt.json
vekt diff v1.safetensors v2.safetensors
Manifest Comparison:
==================
Unchanged Tensors: 289
~ Modified: layers.10.attention.weight (Shape [4096, 4096])
~ Modified: layers.10.mlp.weight (Shape [11008, 4096])
Storage savings: 99.2% deduplication

Zero Redundancy

Fine-tune BERT ten ways, store the encoder exactly once. If a layer hasn't changed, it takes up zero additional space on disk.

Semantic Diff

Don't just see "Binary file changed". See exactly which tensors were modified, added, or removed between model versions.

Remote Sync

Push to S3 compatible storage. vekt push only uploads the specific blobs that the remote is missing.
